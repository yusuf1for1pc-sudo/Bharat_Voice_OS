CRITICAL DESIGN RULES — NEVER BREAK THESE
Pure black background #000000 on every single screen. Not dark gray. Not navy. Pure black.
White is the only foreground color. #FFFFFF for primary text and active elements. #999999 for secondary text. #1A1A1A for card backgrounds. #2A2A2A for card borders.
Zero color anywhere in the UI during normal use. The app is monochrome black and white. The ONLY exception is the edge glow effect which only activates during agent mode and uses a very subtle periwinkle #818CF8 on the left and right screen edges — invisible during all other states.
The dotted ring component is the heart and soul of this app. It must be built as a custom Flutter painter. It is never a static image. It is never a Lottie animation. It draws individual dots in real time using code.
Monospace font for "SPEAK TO BEGIN" text only. Use Space Mono or Courier. Everything else uses Inter.

THE DOTTED RING — EXACT BEHAVIOR SPECIFICATION
This is the most important component. Read this carefully.
What it looks like: Individual white circular dots arranged roughly in a ring shape. The dots are not uniform — they vary in size and opacity. Dots near the top of the ring are larger and more opaque. Dots near the bottom are smaller and more transparent. This creates a halftone depth effect that makes the ring look three-dimensional. The ring is thick — multiple layers of dots create depth, not just a single line of dots.
Idle state (Image 4): The ring is roughly circular but not mathematically perfect. It is slightly uneven — thicker in some places, thinner in others. It breathes very slowly, expanding and contracting about 3% in size on a 3-second cycle. The ring takes up roughly the top 55% of the screen height. It rotates very slowly — one full rotation every 12 seconds. "SPEAK TO BEGIN" text with blinking cursor sits centered inside it.
Active/blob state (Image 2): The ring morphs into a large organic blob. It is not circular at all. It has multiple bumps and indentations — at least 4 to 5 distinct protrusions making it look like a living organism or an amoeba. The blob is large, filling almost the entire width and about 70% of the height. The dots become denser and larger during this state. The morph from circle to blob happens smoothly over 800 milliseconds when the user taps.
Listening state (Image 1): The ring stays in circle form in the top half while the bottom sheet covers the lower half. When the user speaks, individual dots in the ring jump and pulse in response to the voice amplitude. Dots closer to the user's voice direction get larger. The ring breathes faster — 1.5 second cycle instead of 3 seconds.
Processing state: The ring rotates faster — one rotation every 3 seconds instead of 12. It maintains the blob shape. It pulses steadily.
The dot properties that must vary:

Dot size: 2dp minimum to 5dp maximum
Dot opacity: 5% minimum to 100% maximum
Dot density: more dots packed near the top arc, fewer near the bottom arc
The overall thickness of the ring band: approximately 30dp wide, made of overlapping layers of dots at slightly different radii
The ring is NOT a perfect mathematical circle — even in idle state the radius varies by about plus or minus 5% at different angles, giving it an organic handmade quality

EDGE GLOW — ACTIVATES ONLY DURING AGENT MODE
When the backend returns a response with mode equal to agent and the Accessibility Service begins executing actions on another app, activate the edge glow overlay.
The edge glow is a thin strip approximately 6dp wide running along the left edge and right edge of the screen simultaneously. The strip runs vertically for about 60% of the screen height, centered vertically. The color is periwinkle #818CF8 at 25% opacity. The strip has a gradient — transparent at top, full color at center, transparent at bottom.
The glow pulses — it breathes from 20% opacity to 70% opacity on a 2-second cycle while the agent is working. When the agent completes successfully, the glow flashes bright to 100% opacity then fades out completely over 1.5 seconds. When the agent fails, same flash but at the error color #F87171. After fading the glow is completely invisible again.
This uses Android SYSTEM_ALERT_WINDOW overlay drawn on top of everything using a non-touch-intercepting transparent overlay window. Touches pass through it to the app underneath.

FLOATING BUBBLE
A small circle 60dp diameter floats over all apps. Color is dark gray #1A1A1A with a white sparkle/star icon inside. It sits at bottom right of the screen. Draggable to any position. When tapped it opens the listening bottom sheet over whatever app is currently showing.
The bubble has a subtle white glow ring around it — #FFFFFF at 8% opacity. When the agent mode is active, the bubble pulses with the periwinkle glow synced to the edge glow.

BACKEND CONNECTION
The Flutter app connects to a FastAPI backend running at http://10.0.2.2:8000 for emulator or http://YOUR_LAPTOP_IP:8000 for physical device. Store the backend URL in a config file so it can be changed without touching other code.
All voice audio is recorded as WAV, sent to the /process endpoint as multipart form data. The response is JSON containing mode (answer or agent or confirm), title, answer text if answer mode, details array if agent mode, audio_base64 containing the Bulbul TTS response audio, and detected_language.
Play the audio_base64 response immediately on receiving it — decode from base64 to bytes, write to temp file, play using just_audio package.
For agent mode responses, after playing the audio, send the intent_data to the Kotlin Accessibility Service via MethodChannel named bharat_voice_os/accessibility with method name executeAction.

ANDROID NATIVE — ACCESSIBILITY SERVICE
Create BharatAccessibilityService.kt registered in AndroidManifest with BIND_ACCESSIBILITY_SERVICE permission. The service handles three actions for the demo: book_ola_cab which opens Ola and fills destination using resource ID com.olacabs.customer:id/et_pickup_drop, send_whatsapp which deep links to whatsapp://send?phone= with the contact number and message parameters, and check_pm_kisan which opens the browser to the PM Kisan portal URL.
The service communicates with Flutter via MethodChannel. When Flutter sends executeAction with action name and parameters map, the service performs the corresponding action.
The service also has a captureScreen method using the takeScreenshot API available on Android 11 and above which captures the current screen as a bitmap, converts to base64, and returns it to Flutter via a separate MethodChannel call for the vision agent loop.

PERMISSIONS IN ANDROIDMANIFEST
INTERNET, RECORD_AUDIO, SYSTEM_ALERT_WINDOW, BIND_ACCESSIBILITY_SERVICE, FOREGROUND_SERVICE. The overlay window permission SYSTEM_ALERT_WINDOW must be requested at runtime with a direct intent to Settings.ACTION_MANAGE_OVERLAY_PERMISSION because it cannot be granted via the normal permission dialog.

MOCK RESPONSES FOR TESTING
When audio is sent to /process, the backend checks the transcript for keywords and returns the appropriate mock response. If transcript contains "ola" or "cab" return agent mode response for cab booking. If transcript contains "pm kisan" or "kisan" return answer mode with PM Kisan information. If transcript contains "rahul" or "whatsapp" return confirm mode for WhatsApp message. All other transcripts return a generic answer mode response. This allows complete UI testing without any real AI integration.
